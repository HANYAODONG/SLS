# 实现说明

本文档记录两个技术方案的可行性评估、采纳情况、暂缓原因和本次实际落地的代码修改。

## 一、本次评估的方案文件

本次阅读并评估了以下两个方案：

1. `时间注意力改进实现二_可执行版.md`
2. `深度伪造语音检测作品实现过程.md`

处理原则：

- 不破坏当前已经复现成功的 SLS 主模型；
- 不让已下载的 `MMpaper_model.pth` 权重失效；
- 优先落地不影响主评测流程、但能增强展示、分析和应用价值的功能；
- 对依赖重、验证成本高、可能造成复现结果不可比的改动先做记录，不直接改主线代码。

## 二、方案一：时间注意力改进方案

### 2.1 方案内容概述

该方案建议在现有 SLS 模型中加入：

- 普通时间注意力池化；
- 通道引导时间注意力；
- SwiGLU；
- 新分类头；
- `pooling_type` 等新参数；
- 可能建立 `model_cgta.py` 等实验模型文件。

### 2.2 当前处理结论

经项目组讨论后，本方案本轮不再接入代码。

本次已经删除此前新增的 CGTA/时间注意力实验文件，当前项目继续保持：

- 不修改 `model.py`；
- 不修改原始 SLS 模型结构；
- 不新增需要重新训练才能成立的模型层；
- 只基于原始 `MMpaper_model.pth` 做复现、测试和横向应用扩展。

### 2.3 删除的实验性文件

已删除此前为 CGTA 试验新增的文件：

```text
model_cgta.py
main_cgta.py
evaluate_cgta_compat.py
test_cgta_modules.py
scripts/test_cgta_modules.sh
scripts/eval_wild_20000_cgta.sh
scripts/eval_wild_3000_cgta_compat.sh
scripts/eer_wild_3000_cgta_compat.sh
scores/scores_Wild_3000_cgta_compat.txt
```

同时清理了对应的 Python 缓存文件，避免后续搜索或运行时误用旧实验入口。

### 2.4 不接入的原因

该方案不进入当前实现，主要原因如下：

1. **项目组规定不能重新训练**

   时间注意力、CGTA、SwiGLU、新分类头都属于新增模型结构。新增层没有原始 `MMpaper_model.pth` 中的对应参数。若不训练，这些层只能随机初始化，结果不具备可靠解释力。

2. **会破坏原权重的完整适配**

   `MMpaper_model.pth` 是原始 SLS 模型结构的 checkpoint。模型结构变更后，只能部分加载旧权重，无法做到“整体仍使用原始权重文件完成推理”。

3. **容易让实验结论混乱**

   如果新增随机初始化层后直接跑 In-the-Wild，输出结果既不是原模型复现结果，也不是训练后的注意力改进结果。它只能说明代码可运行，不能说明方案有效。

4. **当前项目更适合做横向扩展**

   现阶段更稳的路线是保留主模型不动，围绕数据集、评测、风险清单、网页助手、报告生成、应用场景展示做扩展。

### 2.5 后续记录

该方案本身仍可作为“未来工作”保留在报告中，但应写成：

```text
时间注意力/CGTA 属于需要重新训练的结构级改进方案。由于本项目阶段性要求只能使用原始权重文件，本轮不纳入实现，后续若允许训练新 checkpoint，可作为单独分支继续研究。
```

## 三、方案二：深度伪造语音检测作品方案

### 3.1 方案内容概述

该方案提出两个应用方向：

1. 自媒体声音维权平台；
2. 社交平台深伪内容离线审计引擎。

涉及能力包括：

- WeSpeaker 声纹识别；
- FFmpeg 音画分离；
- Whisper/ASR；
- VLM 画面语义审计；
- LLM 风险报告生成；
- 司法证据固化和时间戳。

### 3.2 当前处理结论

本次继续采纳横向扩展路线，并新增了一组可选扩展模块。

已采纳的是低风险、低依赖、能直接增强当前项目展示能力的部分：

- 将 In-the-Wild 20000 结果接入本地网页助手；
- 泛化实验统计函数，使 DF 和 In-the-Wild 都能统计；
- 新增高风险样本清单生成工具；
- 新增风险清单脚本。
- 新增 FFmpeg 视频预处理适配；
- 新增 WeSpeaker 声纹注册/匹配适配；
- 新增 Whisper ASR 适配；
- 新增 OpenAI-compatible VLM 图像审计适配；
- 新增本地模拟 TSA 时间戳存证；
- 新增多路审计 LLM 汇总报告入口。

仍未直接做成强依赖的是：

- Milvus/向量数据库；
- 真实法律效力 TSA 服务；
- 生产级司法报告系统；
- 完整前后端平台。

### 3.3 暂不采纳部分的原因

1. **新增依赖过多**

   WeSpeaker、Milvus、Whisper、VLM、FFmpeg 都会显著扩大项目依赖和部署复杂度。因此本次做成可选模块和适配入口，不写入核心 `requirements.txt`。

2. **超出当前复现项目边界**

   当前项目核心是音频 deepfake 检测模型复现。声纹识别、视频审计、司法固化属于完整产品系统，需要单独设计后端、存储、权限和合规流程。

3. **缺少必要数据**

   声纹库需要合法采集目标人物原声样本；视频审计需要视频数据和关键帧；司法报告需要更严格的证据链字段。

4. **容易影响主线稳定性**

   如果把这些功能直接混入主模型代码，会让复现实验、产品演示和外部服务依赖缠在一起，后续排错成本会变高。因此本次将横向能力放到 `extensions/` 和 `extension_audit.py`，与主模型保持隔离。

#### 4.4.3 WeSpeaker 声纹注册与匹配适配

新增文件：

```text
extensions/speaker.py
```

能力：

- 调用 WeSpeaker 提取声纹 embedding；
- 将注册声纹保存为 JSONL；
- 对输入音频做 1:N 余弦相似度匹配。

命令示例：

```bash
venv/bin/python extension_audit.py enroll-speaker \
  --name "target_speaker" \
  --audio path/to/clean_voice.wav \
  --enrollment artifacts/speaker/enrollment.jsonl

venv/bin/python extension_audit.py match-speaker \
  --audio path/to/query.wav \
  --enrollment artifacts/speaker/enrollment.jsonl
```

新增依赖：

```text
wespeaker
```

记录位置：

```text
requirements-extensions.txt
```

当前安装状态：

```text
主 venv 未安装；
venv_speaker 隔离环境已安装并通过 import 验证。
```

原因：

- PyPI 当前索引中没有可直接安装的 `wespeaker` 包；
- 从 WeSpeaker 官方 GitHub 安装时，会拉取 `torch>=2.0.0` 和 `torchaudio>=2.0.0`；
- 当前 SLS 复现环境依赖 `torch==1.12.1+cu113`；
- 若强行安装，会破坏主模型复现环境。

因此，本轮不在当前 venv 安装 WeSpeaker，而是使用独立环境：

```bash
bash scripts/setup_speaker_env.sh
```

并通过桥接方式调用：

```bash
SPEAKER_PYTHON=venv_speaker/bin/python \
venv/bin/python extension_audit.py enroll-speaker \
  --name "target_speaker" \
  --audio path/to/clean_voice.wav
```

项目已新增：

```text
requirements-speaker.txt
scripts/setup_speaker_env.sh
extensions/speaker_bridge.py
```

当前 `venv_speaker` 额外处理：

- 补装 `onnxruntime`；
- 添加 torchaudio 兼容补丁；
- 将 WeSpeaker 中不必要的 s3prl frontend 强导入改为可选导入；
- 主环境仍保持 `torch==1.12.1+cu113`、`torchaudio==0.12.1+cu113`、`protobuf==3.20.3`。

暂不安装到主环境的原因：

- 依赖体积较大；
- 可能触发额外模型下载；
- 需要合法采集目标人物原声样本后才有演示意义。
- 当前环境中 `wespeaker` 尚未安装，运行相关命令会给出明确缺依赖提示。
