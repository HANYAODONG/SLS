# 创新实验复原文档

本文档用于说明本次新增的 hybrid 创新实验线如何与原项目隔离，以及后续如果决定不训练、不使用该实验线，如何快速恢复到稳定复现版本。

## 一、当前改动原则

本次新增内容遵循：

- 不修改 `model.py`；
- 不修改 `main.py`；
- 不修改原有 DF / In-the-Wild 复现脚本；
- 不覆盖 `MMpaper_model.pth`；
- 不删除现有 score、key、protocol 文件；
- 所有模型结构创新放在并列实验文件中。

因此，原项目稳定复现入口仍然是：

```bash
python main.py ...
```

原模型评测脚本仍然是：

```bash
bash scripts/eval_df_20000.sh
bash scripts/eval_wild_20000.sh
```

## 二、本次新增的实验性文件

本次新增的模型实验线文件如下：

```text
model_hybrid.py
main_hybrid.py
test_hybrid_modules.py
scripts/test_hybrid_modules.sh
scripts/train_hybrid_stat_sls.sh
scripts/train_hybrid_swiglu.sh
scripts/train_hybrid_temporal.sh
scripts/train_hybrid_full.sh
scripts/eval_hybrid_df_20000.sh
scripts/eval_hybrid_wild_20000.sh
```

这些文件全部属于实验性文件，用于实现：

- Mean+Std 统计感知 SLS；
- SwiGLU 双级门控；
- 普通时间注意力池化；
- 通道引导时间注意力池化；
- hybrid 模型训练、评测和消融实验。

## 三、如果不训练，应该如何处理

如果项目组后续决定不重新训练模型，则不要使用以下入口：

```bash
python main_hybrid.py ...
bash scripts/train_hybrid_*.sh
bash scripts/eval_hybrid_*.sh
```

继续使用原入口即可：

```bash
python main.py ...
bash scripts/eval_df_20000.sh
bash scripts/eval_wild_20000.sh
```

同时在报告中写明：

```text
hybrid 模型结构已经完成代码级接入，但由于新增层需要重新训练获得有效参数，本轮展示仍以原始 SLS 权重 MMpaper_model.pth 的复现结果为主。
```

后续为了支持训练后的 hybrid checkpoint 解释，`model_hybrid.py` 还加入了可选的 `return_details/layer_mask` 接口。该接口默认关闭，不会影响正常训练和评测；如果决定完全弃用 hybrid 实验线，只需继续使用原 `model.py/main.py` 和原评测脚本即可。

## 四、快速复原方案

### 方案 A：不删除文件，只回到原主线

这是推荐方案。保留实验文件，但运行时只使用原入口：

```bash
bash scripts/eval_df_20000.sh
bash scripts/eval_wild_20000.sh
```

优点：

- 不会丢失已经写好的实验代码；
- 后续允许训练时可以继续使用；
- Git 历史保持完整。

### 方案 B：删除实验文件

如果必须让项目目录完全恢复到不含 hybrid 实验线的状态，可以删除：

```text
model_hybrid.py
main_hybrid.py
test_hybrid_modules.py
scripts/test_hybrid_modules.sh
scripts/train_hybrid_stat_sls.sh
scripts/train_hybrid_swiglu.sh
scripts/train_hybrid_temporal.sh
scripts/train_hybrid_full.sh
scripts/eval_hybrid_df_20000.sh
scripts/eval_hybrid_wild_20000.sh
```

删除后需要确认：

```bash
python -m py_compile model.py main.py
bash scripts/eval_df_tiny10.sh
```

### 方案 C：使用 Git 回退

如果这些文件已经提交到 Git，可以通过 Git 查看新增文件并回退对应提交。

查看最近提交：

```bash
git log --oneline -5
```

如果需要撤销某个只包含 hybrid 实验线的提交：

```bash
git revert <commit_sha>
```

不要使用：

```bash
git reset --hard
```

除非已经确认不会丢失本地数据、报告或队友提交。

## 五、权重兼容说明

原始权重：

```text
MMpaper_model.pth
```

只与原始 `model.py` 完整匹配。

本次新增的 hybrid 结构包含新参数，例如：

- `stat_sls.weight_predictor.*`
- `swiglu.*`
- `temporal_attention.*`
- `cgta_pooling.*`
- `hybrid_classifier.*`

这些参数不存在于 `MMpaper_model.pth` 中。

因此：

- 原权重不能作为 hybrid 模型的最终有效权重；
- 可以用 `--load_strict 0` 做部分加载和调试；
- 正式实验必须训练生成新的 checkpoint；
- 新 checkpoint 只能配合 `model_hybrid.py` / `main_hybrid.py` 使用。

## 六、稳定主线检查命令

如果担心实验文件影响原项目，可运行：

```bash
source venv/bin/activate
python -m py_compile model.py main.py data_utils_SSL.py
bash scripts/eval_df_tiny10.sh
```

只要以上命令正常，说明原项目主线仍可用。
