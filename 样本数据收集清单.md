# 样本数据收集清单

本文档只面向样本数据收集，不包含环境安装步骤。收集到的数据建议统一放在 `samples/` 目录下，避免和 `data/` 中的 ASVspoof 数据混在一起。

## 一、最低可演示闭环所需样本

最低闭环：

```text
视频 -> 音频 -> ASR -> 文本风险审计 -> 本地存证 -> LLM 综合报告
```

需要收集：

1. `samples/video/demo_video.mp4`

   要求：

   - mp4 格式；
   - 10 秒到 2 分钟；
   - 有清晰人声；
   - 最好有人脸或人物画面；
   - 文件名不要包含空格；
   - 不涉及隐私和版权风险。

2. `samples/text/demo_transcript.txt`

   要求：

   - 一段可用于内容安全审计的文本；
   - 可以是正常文本，也可以是模拟诈骗、误导、冒充公众人物发言等风险文本；
   - 不要使用真实个人敏感信息。

## 二、声纹功能所需样本

用于验证 WeSpeaker 声纹注册和匹配。

建议目录：

```text
samples/speaker/enroll/
samples/speaker/query/
```

需要收集：

1. 注册音频：

   ```text
   samples/speaker/enroll/target_clean.wav
   ```

   要求：

   - 同一个目标说话人；
   - 20 到 60 秒；
   - 单人说话；
   - 背景噪声尽量低；
   - wav 格式优先；
   - 需要合法授权。

2. 查询音频：

   ```text
   samples/speaker/query/query_same.wav
   samples/speaker/query/query_other.wav
   ```

   要求：

   - `query_same.wav` 尽量来自同一说话人；
   - `query_other.wav` 来自不同说话人；
   - 每段 5 到 30 秒；
   - 用于观察相似度差异。

## 三、VLM 图像审计所需样本

用于验证关键帧/图片是否能被视觉大模型审计。

建议目录：

```text
samples/images/
```

需要收集：

```text
samples/images/frame_public_person.jpg
samples/images/frame_normal_scene.jpg
```

要求：

- jpg/png 格式；
- 至少一张有人物或公众场景；
- 至少一张普通非敏感场景；
- 不要使用侵犯隐私或版权不清晰的图片。

也可以直接使用视频预处理生成的关键帧：

```text
artifacts/audit/demo_video/frames/frame_00001.jpg
```

## 四、音频伪造检测展示样本

当前项目已有 In-the-Wild 数据和 DF 数据，可继续使用。

如果要额外准备演示音频，建议：

```text
samples/audio/real_demo.wav
samples/audio/fake_demo.wav
```

要求：

- 16kHz 单声道 wav 最佳；
- 一段真实人声；
- 一段 AI 合成或伪造人声；
- 每段 5 到 30 秒；
- 不要求加入 ASVspoof 正式评测，只用于产品演示。

## 五、综合报告输入样本

用于测试最终 LLM 风险报告。

建议准备：

```text
samples/audit_payload/demo_payload.json
```

内容示例：

```json
{
  "case_id": "demo_case",
  "audio_detection": {
    "model": "SLS baseline",
    "score": -12.3,
    "judgement": "疑似伪造"
  },
  "asr_audit": {
    "text": "示例转写文本",
    "risk": "待分析"
  },
  "speaker_match": {
    "target": "target_speaker",
    "similarity": 0.92
  },
  "vision_audit": {
    "summary": "画面审计结果"
  },
  "timestamp": {
    "sha256": "demo"
  }
}
```


## 六、注意事项

- 样本尽量短小，先保证流程跑通；
- 不要收集未经授权的个人隐私音频；
- 不要把大型样本提交到 GitHub；
- 样本目录 `samples/` 建议本地保存，必要时单独共享；
- 如果视频太大，先压缩到 1 到 2 分钟以内。
