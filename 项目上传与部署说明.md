# 项目上传与部署说明

本文档说明当前项目上传到 GitHub 的内容范围、部署者需要自行准备的资源、新环境部署步骤、原 SLS 复现流程、hybrid 创新训练流程，以及训练完成后如何使用新的权重文件。

目标仓库：

```text
https://github.com/HANYAODONG/SLS
```

## 一、当前项目状态

当前项目包含三条工作线：

1. **原 SLS 主模型复现线**

   入口文件：

   ```text
   model.py
   main.py
   ```

   使用原始权重：

   ```text
   MMpaper_model.pth
   ```

   已完成结果：

   ```text
   DF 20000            EER = 1.99%
   In-the-Wild 20000   EER ≈ 7.47%
   ```

2. **hybrid 算法创新实验线**

   入口文件：

   ```text
   model_hybrid.py
   main_hybrid.py
   ```

   已接入模块：

   - Mean+Std 统计感知 SLS；
   - SwiGLU 特征门控；
   - 普通时间注意力池化；
   - 通道引导时间注意力池化。

   说明：

   ```text
   hybrid 实验线新增了模型参数，不能直接把 MMpaper_model.pth 当作最终有效权重使用。
   必须重新训练生成 models/.../best.pth 后，才能进行正式评测。
   ```

3. **横向扩展应用线**

   已实现：

   - DeepSeek 本地网页助手；
   - FFmpeg 视频音画分离；
   - Whisper ASR 适配；
   - WeSpeaker 声纹注册与匹配适配；
   - VLM 图像审计接口；
   - 本地模拟 TSA 时间戳存证；
   - 高风险样本清单导出；
   - LLM 综合风险报告入口。

## 四、新环境部署步骤

### 4.1 克隆仓库

```bash
git clone https://github.com/HANYAODONG/SLS.git
cd SLS
```

### 4.2 创建主环境

```bash
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
```

安装 PyTorch：

```bash
pip install torch==1.12.1+cu113 torchvision==0.13.1+cu113 torchaudio==0.12.1+cu113 \
  --extra-index-url https://download.pytorch.org/whl/cu113
```

安装依赖：

```bash
pip install -r requirements.txt
```

安装 fairseq：

```bash
unzip -n fairseq-a54021305d6b3c4c5959ac9395135f63202db8f1.zip
pip install --editable ./fairseq-a54021305d6b3c4c5959ac9395135f63202db8f1
```

如果压缩包不存在但目录已存在：

```bash
pip install --editable ./fairseq-a54021305d6b3c4c5959ac9395135f63202db8f1
```

## 六、hybrid 训练运行

### 6.1 模块测试

```bash
source venv/bin/activate
bash scripts/test_hybrid_modules.sh
```

预期输出：

```text
All hybrid module tests passed.
```

### 6.2 单 epoch 试跑

```bash
EPOCHS=1 BATCH_SIZE=1 bash scripts/train_hybrid_full.sh
```

如果能正常生成 `models/.../epoch_0.pth` 和 `best.pth`，说明训练链路基本打通。

### 6.3 四组实验

```bash
EPOCHS=10 BATCH_SIZE=1 bash scripts/train_hybrid_stat_sls.sh
EPOCHS=10 BATCH_SIZE=1 bash scripts/train_hybrid_swiglu.sh
EPOCHS=10 BATCH_SIZE=1 bash scripts/train_hybrid_temporal.sh
EPOCHS=10 BATCH_SIZE=1 bash scripts/train_hybrid_full.sh
```

含义：

```text
stat_sls    只加 Mean+Std 统计感知 SLS
swiglu      只加 SwiGLU
temporal    只加普通时间注意力
full        Mean+Std + SwiGLU + CGTA
```

### 6.4 早停

默认：

```text
EARLY_STOP_PATIENCE=3
```

如果连续 3 个 epoch 训练 loss 没有改善，会提前停止。

如果想强制跑满：

```bash
EARLY_STOP_PATIENCE=0 EPOCHS=10 bash scripts/train_hybrid_full.sh
```

## 七、训练后如何使用新权重

hybrid 训练输出在：

```text
models/<实验目录>/
```

常见文件：

```text
epoch_0.pth
epoch_1.pth
best.pth
```

正式评测优先使用：

```text
best.pth
```

### 7.1 DF 20000

```bash
MODEL_PATH=models/<实验目录>/best.pth bash scripts/eval_hybrid_df_20000.sh
python evaluate_2021_DF.py scores/scores_Hybrid_DF_20000.txt ./keys eval
```

### 7.2 In-the-Wild 20000

```bash
MODEL_PATH=models/<实验目录>/best.pth bash scripts/eval_hybrid_wild_20000.sh
python evaluate_in_the_wild.py scores/scores_Hybrid_Wild_20000.txt ./keys eval
```

### 7.3 单模块 checkpoint 的评测开关

训练和评测开关必须一致。

例如只加 Mean+Std：

```bash
MODEL_PATH=models/<实验目录>/best.pth \
USE_STAT_SLS=1 \
USE_SWIGLU=0 \
POOLING_TYPE=maxpool \
bash scripts/eval_hybrid_df_20000.sh
```

只加 SwiGLU：

```bash
MODEL_PATH=models/<实验目录>/best.pth \
USE_STAT_SLS=0 \
USE_SWIGLU=1 \
POOLING_TYPE=maxpool \
bash scripts/eval_hybrid_df_20000.sh
```

只加普通时间注意力：

```bash
MODEL_PATH=models/<实验目录>/best.pth \
USE_STAT_SLS=0 \
USE_SWIGLU=0 \
POOLING_TYPE=temporal \
bash scripts/eval_hybrid_df_20000.sh
```

完整模型默认：

```text
USE_STAT_SLS=1
USE_SWIGLU=1
POOLING_TYPE=cgta
```

因此可以直接：

```bash
MODEL_PATH=models/<实验目录>/best.pth bash scripts/eval_hybrid_df_20000.sh
```
