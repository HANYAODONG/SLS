# 项目交接与后续训练指南

本文档用于在更换服务器、重新配置环境、开启新对话或交接给队友时，快速恢复对整个项目的上下文理解。它从项目目标、目录结构、已完成工作、数据与权重、复现流程、创新实验线、训练操作、训练后权重使用、评测和回滚方案几个方面完整说明当前项目状态。

## 一、项目总体目标

本项目基于开源仓库：

```text
SLSforASVspoof-2021-DF
```

当前项目目标分为三层：

1. **原 SLS 主模型复现**

   使用原始 SLS 模型结构、XLS-R 300M 前端和原始权重 `MMpaper_model.pth`，完成 ASVspoof 2021 DF 与 In-the-Wild 数据集评测。

2. **不改变主模型的横向扩展**

   围绕音频深度伪造检测任务，扩展网页助手、风险报告、视频音画分离、ASR、声纹、VLM 审计、时间戳存证等工程能力。

3. **模型结构创新实验**

   参考三个方案文档，新增 hybrid 实验线，实现：

   - Mean+Std 统计感知 SLS；
   - SwiGLU 特征门控；
   - 普通时间注意力池化；
   - 通道引导时间注意力池化。

   该实验线不会覆盖原项目文件。它需要重新训练才能得到有效结果。

4. **零训练可解释性报告**

   参考 `XLS-R_SLS_大模型可解释性零训练可执行方案(1).md`，新增 `explainability/` 模块和 `scripts/explain_audio.py`，在不修改原模型、不重新训练的前提下生成结构化证据和中文模板报告。

## 八、hybrid 创新模型结构

hybrid 实验线放在：

```text
model_hybrid.py
main_hybrid.py
```

### 8.1 Mean+Std 统计感知 SLS

对应类：

```text
StatisticalSLS
```

逻辑：

```text
XLS-R 24 层隐藏特征 [B, L, T, D]
→ 每层计算 mean/std
→ MLP 预测层权重
→ 对 24 层加权求和
→ 得到 [B, T, D]
```

开关：

```bash
--use_stat_sls 1
--stat_sls_use_std 1
```

### 8.2 SwiGLU 特征门控

对应类：

```text
SwiGLUGate
```

逻辑：

```text
fused [B, T, D]
→ LayerNorm
→ SiLU(Wg x) * Wv x
→ out projection
→ residual
```

开关：

```bash
--use_swiglu 1
```

### 8.3 普通时间注意力池化

对应类：

```text
TemporalAttentionPooling
```

开关：

```bash
--pooling_type temporal
```

### 8.4 通道引导时间注意力

对应类：

```text
ChannelGuidedTemporalAttention
```

开关：

```bash
--pooling_type cgta
--cgta_use_std 1
--cgta_stat_residual 1
```

### 8.5 原 head 保留

如果使用：

```bash
--pooling_type maxpool
```

则走类似原模型的后端：

```text
BatchNorm2d → SELU → max_pool2d → fc1 → fc3
```

这用于消融实验，比如只测试 Mean+Std 或只测试 SwiGLU。

## 十一、训练参数说明

训练脚本支持环境变量覆盖：

```text
EPOCHS                 训练轮数，默认 10
BATCH_SIZE             batch size，默认 1
DATABASE_PATH          数据根目录，默认 data
XLSR_CHECKPOINT        XLS-R checkpoint，默认 xlsr2_300m.pt
NUM_WORKERS            DataLoader worker 数，默认 2
EARLY_STOP_PATIENCE    早停耐心值，默认 3
```

示例：

```bash
EPOCHS=3 BATCH_SIZE=1 NUM_WORKERS=2 bash scripts/train_hybrid_full.sh
```

### 11.1 关于早停

当前 `main_hybrid.py` 使用训练 loss 做简单早停：

```text
如果连续 EARLY_STOP_PATIENCE 个 epoch 训练 loss 没有变好，就提前停止。
```

默认：

```text
EARLY_STOP_PATIENCE=3
```

如果希望完整跑满 epoch，可设置：

```bash
EARLY_STOP_PATIENCE=0 EPOCHS=10 bash scripts/train_hybrid_full.sh
```

### 11.2 关于显存

本机曾使用：

```text
NVIDIA GeForce RTX 3050 Laptop GPU
显存 4GB
```

建议：

```text
BATCH_SIZE=1
EVAL_BATCH_SIZE=1
--disable_cudnn
```

如果换到更大显存服务器，可以尝试：

```bash
BATCH_SIZE=2
BATCH_SIZE=4
```

但每次提高 batch size 后，都建议先跑 1 epoch 检查显存。

## 十二、训练输出和 checkpoint

训练后会在 `models/` 下生成目录，命名类似：

```text
models/hybrid_DF_weighted_CCE_10_1_stat1_swiglu1_poolcgta_full/
```

其中包含：

```text
epoch_0.pth
epoch_1.pth
...
best.pth
```

含义：

- `epoch_*.pth`：每个 epoch 后保存的 checkpoint；
- `best.pth`：当前训练 loss 最低的 checkpoint；
- 正式评测优先使用 `best.pth`。

### 12.1 checkpoint 和原 `.pth` 的关系

`MMpaper_model.pth`：

```text
原 SLS 模型权重，只适配 model.py。
```

`models/.../best.pth`：

```text
hybrid 模型训练出来的新权重，只适配 model_hybrid.py。
```

二者都是 PyTorch 权重文件，但模型结构不同，不能混用。

### 12.2 是否可以加载原权重继续训练

`main_hybrid.py` 支持：

```bash
--model_path MMpaper_model.pth --load_strict 0
```

含义：

```text
非严格加载旧权重，能匹配的层加载，新增层随机初始化。
```

注意：

- 这只适合调试或迁移初始化；
- 不能作为最终有效结果；
- 最终仍需要训练生成 hybrid checkpoint。

当前训练脚本默认不加载 `MMpaper_model.pth`。如果后续要做迁移初始化，可以在脚本中加：

```bash
--model_path MMpaper_model.pth --load_strict 0
```

### 13.4 训练后生成 hybrid 可解释性报告

`model_hybrid.py` 已经预留可解释性接口：

```text
forward(x, return_details=False, layer_mask=None)
```

默认 `return_details=False` 时仍返回原来的模型输出，训练和评测脚本不需要改。
当 `return_details=True` 时，会额外返回：

```text
hidden_states
fused_sequence
embedding
layer_weights
effective_layer_weights
temporal_weights
channel_context
log_probabilities
```

其中 `layer_mask` 用于解释性实验中的层遮挡/层消融，会在保持权重总和的前提下重新归一化。这个接口只服务 hybrid 实验线，不影响原 `model.py`。

训练得到 hybrid checkpoint 后，可以用同一套解释脚本，只是增加 `--model-type hybrid`：

```bash
python scripts/inspect_model_output.py \
  --model-type hybrid \
  --audio release_in_the_wild/0.wav \
  --checkpoint models/<hybrid实验目录>/best.pth \
  --xlsr-checkpoint xlsr2_300m.pt \
  --device cpu
```

```bash
python scripts/compare_original_output.py \
  --model-type hybrid \
  --audio release_in_the_wild/0.wav \
  --checkpoint models/<hybrid实验目录>/best.pth \
  --xlsr-checkpoint xlsr2_300m.pt \
  --device cpu
```

```bash
python scripts/explain_audio.py \
  --model-type hybrid \
  --audio release_in_the_wild/0.wav \
  --checkpoint models/<hybrid实验目录>/best.pth \
  --xlsr-checkpoint xlsr2_300m.pt \
  --config configs/explainability_cpu_debug.json \
  --output-dir artifacts/reports \
  --device cpu
```

如果解释的是单模块 checkpoint，必须保持解释时开关和训练时一致。例如只训练 `stat_sls` 时，需要额外带上：

```bash
--use-stat-sls --pooling-type maxpool
```

完整 hybrid 模型默认等价于：

```text
--use-stat-sls
--use-swiglu
--pooling-type cgta
```
