# 深度伪造语音检测作品实现

---

## 1. 作品一：自媒体声音维权平台（针对创作者/MCN）

### 1.1 技术架构图
*   **前端**：React/Vue 构建的维权工作台（上传音频、查看报告、声纹入库）。
*   **后端**：FastAPI/Flask 处理业务逻辑。
*   **算法层**：
    *   **WeSpeaker (开源)**：负责 1:N 声纹匹配。
    *   **自主研发模型**：负责 Deepfake 物理特征检测。
    *   **大模型 (LLM)**：负责生成司法级鉴定描述。

### 1.2 核心实现细节（分步骤）

#### 第一步：构建“定向原声库”（Enrollment）
*   **操作**：爬取 5-10 位头部博主（如罗翔、董宇辉等）的 30 秒纯净人声。
*   **实现**：使用 **WeSpeaker** 提取声纹嵌入向量（Embedding），存储在向量数据库（如 Milvus 或简单的 Numpy 数组）中。
*   **代码思路**：
    ```python
    import wespeaker
    model = wespeaker.load_model('chinese')
    embedding = model.extract_embedding('boss_voice.wav')
    # 存储 embedding 到库中，关联博主姓名
    ```

#### 第二步：可疑音频匹配与真伪判定（Verification）
1.  **身份锁定**：用户上传音频，WeSpeaker 计算其与库中所有博主的余弦相似度，选出得分最高的博主。
2.  **真伪裁决**：**调用您的核心算法**，判定是否为“AI伪造”。

#### 第三步：司法证据固化（Certification）
*   **时间戳**：调用 TSA 模拟接口，为报告分配唯一 ID 和时间戳。
*   **报告生成**：将检测得分、频谱图、匹配博主信息发送给 LLM（如 GPT-4o），Prompt 如下：
    > “请作为一名司法鉴定专家，根据以下数据撰写报告：声纹相似度 98%，伪造概率 99%，目标人物罗翔。指出伪造点在于基频轨迹过于平滑，不符合自然发声规律。”

## 2. 作品二：社交平台“深伪内容离线审计”引擎（针对平台方）

### 2.1 技术架构图
*   **数据流**：模拟视频流 -> 音画分离 -> 多路并行审计 -> 综合风险评估报告。
*   **核心组件**：
    *   **FFmpeg**：多媒体处理。
    *   **多模态大模型 (VLM)**：画面语义审计。
    *   **自主研发模型**：音频物理层检测。
    *   **LLM**：逻辑汇总与风险分级。

### 2.2 核心实现细节（分步骤）

#### 第一步：音画分离与预处理
*   **操作**：使用 FFmpeg 提取音频（.wav）和关键帧（.jpg）。
    ```bash
    ffmpeg -i input_video.mp4 -vn -acodec pcm_s16le -ar 16000 audio.wav
    ffmpeg -i input_video.mp4 -vf "select=eq(pict_type\,I)" -vsync vfr frames/%03d.jpg
    ```

#### 第二步：多路并行审计（The Multi-path Audit）
1.  **路径 A（画面审计 - VLM）**：将关键帧发给 Qwen-VL 或 GPT-4o。
    *   **Prompt**： “识别图中人物是谁？是否涉及重大新闻、政治或突发灾害场景？请回答‘敏感’或‘非敏感’。”
2.  **路径 B（音频内容审计 - ASR + LLM）**：
    *   使用 **Whisper** 将音频转为文字。
    *   **Prompt**： “分析以下文本是否包含虚假指令、诈骗诱导或敏感言论：[ASR文本]。”
3.  **路径 C（音频物理检测 - 核心算法）**：
    *   **调用您的算法**，给出 0-1 的伪造概率分值。

#### 第三步：风险分类与自动预警（Final Assessment）
*   **逻辑汇总**：将三路结果汇总给 LLM 做决策。
    *   **输入**：画面敏感=是，内容敏感=是，伪造概率=0.92。
    *   **LLM 决策**： “**判定为极高风险（Red Alert）**。原因：该内容伪造了公众人物在敏感场景下的发言，且语音检测证实为 AI 合成。建议：立即下架并人工复核。”
