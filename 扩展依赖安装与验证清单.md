# 扩展依赖安装与验证清单

本文档面向部署者，记录横向扩展功能的依赖安装和验证命令。

## 一、主环境状态

主环境：

```text
venv/
```

当前核心依赖应保持：

```text
torch==1.12.1+cu113
torchaudio==0.12.1+cu113
protobuf==3.20.3
```

验证：

```bash
cd ~/Projects/SLSforASVspoof-2021-DF
source venv/bin/activate
python - <<'PY'
import torch, torchaudio, google.protobuf
print(torch.__version__)
print(torchaudio.__version__)
print(google.protobuf.__version__)
PY
pip check
```

## 二、系统依赖

安装 FFmpeg：

```bash
sudo apt-get update
sudo apt-get install ffmpeg
```

验证：

```bash
ffmpeg -version
ffprobe -version
```

## 三、主环境可安装依赖

这些可以安装在主 `venv`：

```bash
source venv/bin/activate
pip install -r requirements-extensions.txt
```

当前包含：

```text
imageio-ffmpeg
openai-whisper
```

验证：

```bash
python -c "import whisper; print('whisper ok')"
python -c "import imageio_ffmpeg; print(imageio_ffmpeg.get_ffmpeg_version())"
```

## 四、WeSpeaker 独立环境

不要把 WeSpeaker 安装到主 `venv`。

安装：

```bash
cd ~/Projects/SLSforASVspoof-2021-DF
bash scripts/setup_speaker_env.sh
```

验证：

```bash
venv_speaker/bin/python -c "import wespeaker; print('wespeaker ok')"
bash scripts/check_extension_deps.sh
```

当前应看到：

```text
[ok] speaker env module: wespeaker (venv_speaker/bin/python)
```

说明：

- `venv_speaker` 使用独立的 `torch/torchaudio/protobuf`；
- 不影响主 SLS 复现环境；
- 安装脚本会补 `onnxruntime`；
- 安装脚本会加 torchaudio 兼容补丁；
- 安装脚本会禁用不必要的 s3prl frontend 强导入。

## 五、Milvus / pymilvus

当前不在本项目主环境中安装。

原因：

```text
pymilvus 会要求 protobuf>=5，和主项目 protobuf==3.20.3 冲突。
```

小规模声纹演示直接使用：

```text
artifacts/speaker/enrollment.jsonl
```

如果后续必须使用 Milvus，建议单独服务化：

```bash
python3 -m venv venv_milvus
source venv_milvus/bin/activate
pip install pymilvus
```

## 六、总体依赖检查

运行：

```bash
bash scripts/check_extension_deps.sh
```

当前理想状态：

```text
[ok] command: ffmpeg
[ok] command: ffprobe
[ok] python fallback: imageio_ffmpeg
[missing] python module: wespeaker
[ok] speaker env module: wespeaker (venv_speaker/bin/python)
[ok] python module: whisper
[missing] python module: pymilvus
```

其中：

- 主环境 `wespeaker` missing 是正常的；
- `speaker env module: wespeaker` ok 表示声纹功能可通过隔离环境桥接；
- `pymilvus` missing 是当前设计选择，不影响 JSONL 小规模声纹库。

## 七、功能验证命令

### 7.1 视频预处理

```bash
CASE_ID=demo_video bash scripts/preprocess_video_demo.sh samples/video/demo_video.mp4
```

### 7.2 本地存证

```bash
bash scripts/certify_file_demo.sh README.md
```

### 7.3 Whisper ASR

```bash
venv/bin/python extension_audit.py asr \
  --audio artifacts/audit/demo_video/audio.wav \
  --model-name base \
  --output artifacts/audit/demo_video/asr.json
```

### 7.4 文本风险审计

```bash
venv/bin/python extension_audit.py audit-text \
  --text-file samples/text/demo_transcript.txt \
  --output artifacts/audit/demo_video/text_audit.json
```

### 7.5 声纹注册

```bash
SPEAKER_PYTHON=venv_speaker/bin/python \
venv/bin/python extension_audit.py enroll-speaker \
  --name "target_speaker" \
  --audio samples/speaker/enroll/target_clean.wav \
  --enrollment artifacts/speaker/enrollment.jsonl
```

### 7.6 声纹匹配

```bash
SPEAKER_PYTHON=venv_speaker/bin/python \
venv/bin/python extension_audit.py match-speaker \
  --audio samples/speaker/query/query_same.wav \
  --enrollment artifacts/speaker/enrollment.jsonl
```

### 7.7 VLM 图像审计

需要额外准备视觉模型 API：

```bash
VLM_BASE_URL="https://your-vlm-endpoint/v1" \
VLM_API_KEY="your_vlm_key" \
VLM_MODEL="your_vision_model" \
venv/bin/python extension_audit.py audit-image \
  --image samples/images/frame_public_person.jpg \
  --output artifacts/audit/demo_video/frame_audit.json
```

### 7.8 综合报告

```bash
venv/bin/python extension_audit.py report \
  --input samples/audit_payload/demo_payload.json \
  --output artifacts/audit/demo_video/final_report.json
```

## 八、不要执行的命令

不要在主 `venv` 中执行：

```bash
pip install git+https://github.com/wenet-e2e/wespeaker.git
pip install pymilvus
```

原因：

- 会破坏主环境的 `torch` 或 `protobuf`；
- 影响 SLS 主模型复现稳定性。
