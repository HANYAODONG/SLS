# 声纹相似度 × 伪造概率联合判别矩阵——实现说明

> 本文档说明"定向原声库"方向的核心创新点：将 WeSpeaker 声纹比对结果与本作品深度伪造检测模型的输出进行融合判别，构建二维风险矩阵。该模块为独立后处理层，不修改原有 `model.py`、`model_hybrid.py` 及现有声纹比对逻辑，符合本项目"隔离改动"的开发原则。

## 一、设计动机

### 1.1 现有流程的局限

在"定向原声库"应用场景中，原有流程是：

```text
WeSpeaker 声纹匹配 → 锁定身份（是谁说的）
伪造检测模型      → 独立判断真假（是不是 AI 生成）
```

两个判断彼此独立、串行执行，结果分别展示，互不影响对方的置信度。这带来的问题是：仅看伪造概率，无法区分"普通人随口编造的内容"和"精准模仿目标人物声音的深度伪造"，而后者在维权场景下危害显著更大、处理也更紧迫。

### 1.2 创新思路

将声纹相似度作为伪造判定的**辅助风险信号**，而非单纯的身份标签，构建声纹相似度与伪造概率的二维联合判别矩阵，使系统能够自动识别"高仿冒 + 高伪造"这一类最具危害性的攻击组合，输出统一的风险等级，而不是把两个独立数字简单堆叠展示。

## 二、判别矩阵设计

矩阵以校准后的声纹相似度为纵轴、校准后的伪造概率为横轴，划分为四个象限：

| | 伪造概率低 | 伪造概率高 |
|---|---|---|
| **声纹相似度高** | 区域①：低风险——疑似本人真实发言 | 区域②：高危——精准仿冒 + AI 合成 |
| **声纹相似度低** | 区域③：低风险——非目标人物，非合成 | 区域④：中危——冒用身份，仿真度有限 |

四个象限对应不同的风险等级与处理建议，便于在前端以可视化散点图形式呈现，用户上传样本对应的点落在哪个象限一目了然。

## 三、实现流程

整体实现分为四个步骤：分数校准 → 阈值确定 → 矩阵判别 → 系统集成。模块作为后处理层接入，不需要重新训练神经网络，工作量可控。

```text
WeSpeaker 声纹模型 ──→ voice_similarity (原始余弦相似度)
                                              ↘
                                                联合判别模块 ──→ risk_level / risk_description / quadrant
                                              ↗
深度伪造检测模型   ──→ fake_probability (原始伪造概率)
```

### 3.1 第一步：分数校准

**问题**：WeSpeaker 输出的余弦相似度和深度伪造检测模型输出的 sigmoid 概率，量纲和分布形态不同。例如真实语音的伪造概率可能集中在 0.05～0.15，而同一人的声纹相似度可能集中在 0.7～0.95，若直接用原始数值定义"高/低"，两者的 0.85 含义并不对等，会导致矩阵判别失真。

**方法**：在验证集上分别统计两个分数的分布，使用百分位数映射进行归一化，而非直接使用原始值。

```python
import numpy as np

class ScoreCalibrator:
    """
    基于验证集分布，将原始分数映射为其在该分布中的百分位（0~1）。
    校准参数（百分位数组）可预先计算并持久化为 json/npy 文件，
    部署时直接加载，无需每次重新统计。
    """

    def __init__(self, voice_sim_scores, fake_prob_scores):
        # voice_sim_scores: 验证集上"同一人"样本对的声纹相似度集合
        # fake_prob_scores: 验证集上全部样本的伪造概率集合
        self.voice_percentiles = np.percentile(voice_sim_scores, np.arange(0, 101, 1))
        self.fake_percentiles = np.percentile(fake_prob_scores, np.arange(0, 101, 1))

    def calibrate_voice(self, raw_score):
        return np.searchsorted(self.voice_percentiles, raw_score) / 100.0

    def calibrate_fake(self, raw_score):
        return np.searchsorted(self.fake_percentiles, raw_score) / 100.0
```

### 3.2 第二步：阈值确定

**问题**：矩阵的两条分界线（声纹阈值、伪造阈值）不应主观拍定，需要有数据依据。

**方法**：借鉴语音伪造检测领域常用的 EER（等错误率）思路——阈值取在误接受率（FAR）与误拒绝率（FRR）相等的工作点，分别应用于声纹相似度和伪造概率两组数据。

```python
def find_eer_threshold(scores_genuine, scores_impostor):
    """
    scores_genuine: 正样本（真实匹配 / 真实语音）的分数集合
    scores_impostor: 负样本（错误匹配 / 伪造语音）的分数集合
    返回 EER 对应的阈值
    """
    thresholds = np.linspace(0, 1, 1000)
    far_list, frr_list = [], []
    for t in thresholds:
        far = np.mean(scores_impostor >= t)
        frr = np.mean(scores_genuine < t)
        far_list.append(far)
        frr_list.append(frr)
    far_arr, frr_arr = np.array(far_list), np.array(frr_list)
    eer_idx = np.argmin(np.abs(far_arr - frr_arr))
    return thresholds[eer_idx]
```

分别在声纹相似度数据和伪造概率数据上调用该函数，得到 `voice_threshold` 与 `fake_threshold`，作为矩阵划分四象限的分界线。该方式使阈值有明确的数据支撑，便于在文档中说明依据。

### 3.3 第三步：矩阵判别逻辑

采用规则判定而非训练新的融合模型，原因是现阶段样本规模不足以支撑单独训练一个可靠的融合分类器；规则判定逻辑清晰、可解释性强，也更便于在答辩中说明判断依据。

```python
class JointRiskAssessment:
    def __init__(self, calibrator, voice_threshold, fake_threshold):
        self.calibrator = calibrator
        self.voice_threshold = voice_threshold
        self.fake_threshold = fake_threshold

    def assess(self, raw_voice_sim, raw_fake_prob):
        voice_score = self.calibrator.calibrate_voice(raw_voice_sim)
        fake_score = self.calibrator.calibrate_fake(raw_fake_prob)

        high_voice = voice_score >= self.voice_threshold
        high_fake = fake_score >= self.fake_threshold

        if high_voice and high_fake:
            level, desc, quadrant = (
                "高危",
                "声纹高度匹配目标人物，且检测为AI合成语音，疑似精准仿冒攻击，"
                "建议立即预警并人工复核",
                "Q2",
            )
        elif high_voice and not high_fake:
            level, desc, quadrant = (
                "低风险",
                "声纹匹配目标人物，且未检测到明显伪造特征，疑似本人真实发言",
                "Q1",
            )
        elif not high_voice and high_fake:
            level, desc, quadrant = (
                "中危",
                "未匹配到目标人物声纹，但检测为AI合成语音，可能为冒用其他身份"
                "或仿真度有限的伪造内容",
                "Q4",
            )
        else:
            level, desc, quadrant = (
                "低风险",
                "未匹配目标人物声纹，且未检测到明显伪造特征",
                "Q3",
            )

        return {
            "risk_level": level,
            "risk_description": desc,
            "quadrant": quadrant,
            "calibrated_voice_score": round(voice_score, 3),
            "calibrated_fake_score": round(fake_score, 3),
        }
```

### 3.4 第四步：系统集成

在服务处理层（对应接口 `/api/detect`）的原有调用链"音频预处理 → XLS-R 特征提取 → SLS/hybrid 分类 → 返回伪造概率"之后，并联接入声纹比对结果：

```python
def detect_pipeline(audio_path):
    # 原有伪造检测流程（不修改）
    fake_prob = run_deepfake_detection(audio_path)  # model.py 或 model_hybrid.py

    # 新增：声纹比对
    voice_embedding = wespeaker_model.extract_embedding(audio_path)
    matched_person, voice_sim = match_against_library(voice_embedding, voice_library)

    # 新增：联合判别
    joint_result = joint_assessor.assess(voice_sim, fake_prob)

    return {
        "fake_probability": fake_prob,
        "matched_person": matched_person,
        "voice_similarity": voice_sim,
        **joint_result,
    }
```

原有伪造检测主线（`model.py` / `model_hybrid.py`）及声纹比对逻辑不受影响，新模块作为独立后处理步骤插入，符合既有的隔离开发策略。

## 四、与可解释性报告的衔接

联合判别结果可作为结构化字段传入既有的 LLM 报告生成环节，使生成的鉴定描述论证链条更完整：

> 原 Prompt 示例：声纹相似度 98%，伪造概率 99%，目标人物 X。
>
> 优化后 Prompt 示例：声纹相似度 98%（高仿冒），伪造概率 99%（高伪造），联合风险等级：高危——精准仿冒 + AI 合成。

## 五、文档撰写建议

建议在文档**第四章 创新性说明**中单独成节，按"问题—方法—价值"结构呈现：

- **问题**：现有方案中身份识别与伪造检测彼此独立，无法识别"高度仿冒目标人物声音"这一类最具危害性的攻击场景；
- **方法**：构建声纹相似度与伪造概率的二维联合判别矩阵，将两个独立信号融合为统一的风险等级输出；
- **价值**：相比单一伪造概率判断，能更精准地识别和预警"精准仿冒 + AI 合成"类高危内容，更贴合维权场景下"谁被冒充、冒充得有多像"的实际诉求。

对应实验验证建议放入**第三章 3.5 消融实验与改进模块验证**，对比"仅用伪造概率判定"与"联合矩阵判定"在同一批样本上的判别差异，体现该模块的有效性。

## 六、注意事项

- 阈值与校准参数依赖验证集分布统计得出，若后续更换数据集或声纹模型，需重新执行第 3.1、3.2 步骤；
- 若暂不具备足够验证集样本支撑 EER 阈值计算，文档中应如实写明"阈值基于小规模验证集初步估计，后续需在更大数据集上进一步校准"，不宜夸大其严谨性；
- 该模块不改变原有伪造检测模型结构，可与原 SLS 主线或 hybrid 实验线并行使用，互不冲突。
