# SLSforASVspoof-2021-DF 复现进度总结

日期：2026-06-12

## 1. 复现范围

本项目目前复现的是论文 **Audio Deepfake Detection with XLS-R and SLS classifier** 在 ASVspoof 2021 DF 任务上的主模型推理与评估流程。

当前复现重点包括：

- XLS-R 300M 特征提取器
- 论文作者发布的 SLS 分类器 checkpoint
- ASVspoof 2021 DF 评估集音频子集
- ASVspoof 2021 DF 官方 keys
- 使用仓库自带评估脚本计算 EER

目前，DF 主模型的推理和 EER 评估链路已经端到端跑通。

## 2. 环境状态

当前使用的 Python 环境是项目目录下的 `venv`。

关键安装与运行组件如下：

- Python 环境：`venv/`
- PyTorch：`1.12.1+cu113`
- torchvision：`0.13.1`
- torchaudio：`0.12.1`
- fairseq：从本地解压后的 fairseq 源码安装
- librosa：`0.9.1`
- numpy：`1.23.5`
- protobuf：`3.20.3`

原始 README 推荐使用 Python 3.7 和 conda 环境，但本次复现已经在当前项目自带的虚拟环境中跑通。

用户终端侧已经确认 CUDA 可用：

- GPU：NVIDIA GeForce RTX 3050 Laptop GPU
- PyTorch 可以看到 CUDA：是
- `torch.cuda.is_available()`：`True`

由于当前 GPU 显存为 4 GB，评估脚本采用以下低显存稳定配置：

- `--eval_batch_size 1`
- `--disable_cudnn`
- `PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb:128`

这些设置用于规避早期测试中遇到的 cuDNN 初始化失败和显存压力问题。

## 6. 为复现所做的代码改动

本次复现过程中进行了若干兼容性和易用性修复。

### `model.py`

- 新增 `--xlsr_checkpoint` 参数，可以配置 XLS-R checkpoint 路径。
- 当 `xlsr2_300m.pt` 缺失时，给出清晰的报错提示。

### `data_utils_SSL.py`

- 新增优先使用 `torchaudio` 的音频读取逻辑。
- 如果 `torchaudio` 读取失败，再回退到 `librosa`。
- 这样修改是因为当前环境中 `librosa/soundfile` 读取官方 FLAC 文件会失败，而 `torchaudio.load` 可以正常读取。

### `main.py`

- 新增 `--eval_batch_size` 参数。
- 新增 `--disable_cudnn` 参数。
- 评估输出从追加写入改为覆盖写入，避免旧结果和新结果混在一起。
- 评估阶段加入 `torch.no_grad()`，降低显存占用。
- 修复训练函数中 optimizer 变量使用不一致的问题。
- 改善训练数据路径拼接方式。
- 修改 `.pth` 加载逻辑：先把 checkpoint 加载到 CPU，再载入模型，避免 GPU 上短时间出现双份权重导致显存峰值过高。

### `evaluate_2021_DF.py`

- 支持多种 key 目录布局。
- 除 `hidden_track` 外，也支持官方 metadata 中的 `hidden` phase 命名。

### 脚本

新增了以下可直接运行的脚本：

```text
scripts/eval_df_tiny10.sh
scripts/eval_df_5000.sh
scripts/eer_df_5000.sh
scripts/eval_df_20000.sh
scripts/eer_df_20000.sh
scripts/eval_df_subset.sh
```

EER 脚本现在默认使用：

```text
venv/bin/python
```

这样即使当前 shell 没有直接暴露 `python` 命令，也可以正常运行。

## 12. 注意事项

- 当前 20000 条结果是子集结果，不是完整 ASVspoof 2021 DF benchmark 结果。
- `scores/scores_DF_subset.txt` 来自一次中断的大规模运行，不应作为报告结果使用。
- 对于 4 GB 显存的 RTX 3050 Laptop GPU，建议保持 `EVAL_BATCH_SIZE=1`。
- 如果显存允许，可以测试更大的 batch size，但当前脚本优先保证稳定性。
- 当前 score 取自 `batch_out[:, 1]`，与仓库原始评估流程保持一致。
