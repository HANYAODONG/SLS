# 扩展功能依赖与样本清单

本文档记录横向扩展功能还需要安装的依赖、推荐安装方式，以及验证每个新增功能时需要准备的样本数据。

当前原则：

- 不修改 `model.py`；
- 不改变原始 SLS 模型结构；
- 不破坏当前主复现环境 `venv`；
- 重依赖优先放到独立环境或外部服务中。

## 一、当前已具备的依赖状态

在项目根目录运行：

```bash
cd ~/Projects/SLSforASVspoof-2021-DF
source venv/bin/activate
bash scripts/check_extension_deps.sh
```

当前状态应类似：

```text
[ok] command: ffmpeg
[ok] command: ffprobe
[ok] python fallback: imageio_ffmpeg
[missing] python module: wespeaker
[missing] speaker env module: wespeaker
[ok] python module: whisper
[missing] python module: pymilvus
```

说明：

- `ffmpeg/ffprobe` 已可用；
- `openai-whisper` 已安装；
- `imageio-ffmpeg` 已安装；
- `wespeaker` 不安装在主 `venv`；
- `pymilvus` 不安装在主 `venv`。

## 二、剩余依赖安装命令

### 2.1 FFmpeg/FFprobe

如果检查结果显示 `ffmpeg` 或 `ffprobe` 缺失，执行：

```bash
sudo apt-get update
sudo apt-get install ffmpeg
```

验证：

```bash
ffmpeg -version
ffprobe -version
bash scripts/check_extension_deps.sh
```

备注：

- 目前本机已经安装成功；
- 如果 apt 报 404，先执行 `sudo apt-get update`；
- 如果提示权限不足，说明命令缺少 `sudo`。

### 2.2 Whisper

Whisper 当前已安装在主 `venv`。

如果后续换机器，需要执行：

```bash
cd ~/Projects/SLSforASVspoof-2021-DF
source venv/bin/activate
pip install openai-whisper imageio-ffmpeg
```

或：

```bash
pip install -r requirements-extensions.txt
```

验证：

```bash
python -c "import whisper; print('whisper ok')"
```

注意：

- 第一次实际转写时可能会下载 Whisper 模型权重；
- 建议先用 `base` 或 `small` 模型验证流程；
- CPU 转写会比较慢。

### 2.3 WeSpeaker

不要安装到主 `venv`。

原因：

- WeSpeaker 当前 GitHub 安装链会拉取 `torch>=2` 和 `torchaudio>=2`；
- 当前 SLS 主环境依赖 `torch==1.12.1+cu113`；
- 强行安装可能破坏主模型复现环境。

推荐创建独立环境：

```bash
cd ~/Projects/SLSforASVspoof-2021-DF
bash scripts/setup_speaker_env.sh
```

验证：

```bash
venv_speaker/bin/python -c "import wespeaker; print('wespeaker ok')"
bash scripts/check_extension_deps.sh
```

使用时通过桥接环境变量调用：

```bash
SPEAKER_PYTHON=venv_speaker/bin/python \
venv/bin/python extension_audit.py enroll-speaker \
  --name "target_speaker" \
  --audio path/to/clean_voice.wav \
  --enrollment artifacts/speaker/enrollment.jsonl
```

匹配测试：

```bash
SPEAKER_PYTHON=venv_speaker/bin/python \
venv/bin/python extension_audit.py match-speaker \
  --audio path/to/query.wav \
  --enrollment artifacts/speaker/enrollment.jsonl
```

### 2.4 Milvus / pymilvus

当前不建议装到主 `venv`。

原因：

- `pymilvus` 新版本要求 `protobuf>=5`；
- 当前项目锁定 `protobuf==3.20.3`；
- 直接安装会破坏主复现环境。

短期替代方案：

```text
使用 JSONL + numpy 做小规模声纹库。
```

当前 `extensions/speaker.py` 已经支持 JSONL 形式的 enrollment 文件，不需要 Milvus 也能做小规模 1:N 匹配。

如果后续必须使用 Milvus，建议单独服务化：

```bash
python3 -m venv venv_milvus
source venv_milvus/bin/activate
pip install pymilvus
```

Milvus 服务本身建议用 Docker 部署，不要塞进主复现环境。

## 三、验证新增功能需要准备的样本数据

### 3.1 视频音画分离

功能：

```text
视频 -> wav 音频 + jpg 关键帧
```

需要样本：

```text
一个 mp4 视频文件
```

建议样本规格：

- 时长 10 秒到 2 分钟；
- 有清晰人声；
- 最好包含人脸或人物画面；
- 文件名不要包含空格，避免命令行转义麻烦。

示例路径：

```text
samples/video/demo_video.mp4
```

运行：

```bash
CASE_ID=demo_video bash scripts/preprocess_video_demo.sh samples/video/demo_video.mp4
```

输出：

```text
artifacts/audit/demo_video/audio.wav
artifacts/audit/demo_video/frames/
artifacts/audit/demo_video/preprocess.json
```

检查：

```bash
ls artifacts/audit/demo_video
ls artifacts/audit/demo_video/frames | head
```

### 3.2 Whisper ASR 转写

功能：

```text
音频 -> 文本转写
```

需要样本：

```text
一个包含人声的 wav/mp3/flac 音频
```

可直接使用视频分离得到的：

```text
artifacts/audit/demo_video/audio.wav
```

运行：

```bash
venv/bin/python extension_audit.py asr \
  --audio artifacts/audit/demo_video/audio.wav \
  --model-name base \
  --output artifacts/audit/demo_video/asr.json
```

注意：

- 首次运行会下载 Whisper `base` 模型；
- CPU 下可能较慢；
- 中文/英文都可测试，但清晰人声效果更好。

### 3.3 文本内容风险审计

功能：

```text
ASR 文本 -> DeepSeek 风险分析
```

需要样本：

```text
一段文本，或 ASR 输出文本文件
```

可以手动创建：

```bash
mkdir -p samples/text
echo "请尽快转账到这个账户，否则你的账号会被冻结。" > samples/text/demo_transcript.txt
```

运行：

```bash
venv/bin/python extension_audit.py audit-text \
  --text-file samples/text/demo_transcript.txt \
  --output artifacts/audit/demo_video/text_audit.json
```

需要：

```text
.env 中已配置 DeepSeek API Key
```

### 3.4 VLM 图像审计

功能：

```text
关键帧图片 -> 视觉大模型判断画面风险
```

需要样本：

```text
一张 jpg/png 图片，最好来自视频关键帧
```

示例：

```text
artifacts/audit/demo_video/frames/frame_00001.jpg
```

还需要外部 VLM API：

```text
VLM_BASE_URL
VLM_API_KEY
VLM_MODEL
```

示例命令：

```bash
VLM_BASE_URL="https://your-vlm-endpoint/v1" \
VLM_API_KEY="your_vlm_key" \
VLM_MODEL="your_vision_model" \
venv/bin/python extension_audit.py audit-image \
  --image artifacts/audit/demo_video/frames/frame_00001.jpg \
  --output artifacts/audit/demo_video/frame_audit.json
```

注意：

- DeepSeek 当前配置主要用于文本问答；
- 图像审计需要支持图片输入的视觉模型接口；
- 若没有 VLM API，可先跳过此项。

### 3.5 声纹注册与匹配

功能：

```text
目标人物原声注册 -> 查询音频声纹匹配
```

需要样本：

1. 注册音频：

```text
目标人物 30 秒左右清晰原声
```

2. 查询音频：

```text
待检测音频，最好也是同一说话人或相似说话人
```

建议目录：

```text
samples/speaker/enroll/target_clean.wav
samples/speaker/query/query.wav
```

先安装独立声纹环境：

```bash
bash scripts/setup_speaker_env.sh
```

注册：

```bash
SPEAKER_PYTHON=venv_speaker/bin/python \
venv/bin/python extension_audit.py enroll-speaker \
  --name "target_speaker" \
  --audio samples/speaker/enroll/target_clean.wav \
  --enrollment artifacts/speaker/enrollment.jsonl
```

匹配：

```bash
SPEAKER_PYTHON=venv_speaker/bin/python \
venv/bin/python extension_audit.py match-speaker \
  --audio samples/speaker/query/query.wav \
  --enrollment artifacts/speaker/enrollment.jsonl
```

注意：

- 需要合法获取目标人物样本；
- 背景噪声越小越好；
- 小样本演示可不用 Milvus，JSONL 足够。

### 3.6 本地模拟存证

功能：

```text
文件 -> sha256 + uuid + UTC 时间戳 JSON
```

需要样本：

```text
任意文件
```

示例：

```bash
bash scripts/certify_file_demo.sh artifacts/audit/demo_video/audio.wav
```

输出：

```text
artifacts/audit/demo_case/timestamp.json
```

注意：

- 当前是本地模拟存证；
- 可用于项目展示；
- 不具备正式法律效力。

### 3.7 综合风险报告

功能：

```text
多路审计 JSON -> DeepSeek 综合中文风险报告
```

需要样本：

```text
一个包含多路审计结果的 JSON 文件
```

示例文件：

```json
{
  "case_id": "demo_video",
  "audio_detection": {
    "model": "SLS baseline",
    "score_file": "scores/scores_Wild_20000.txt",
    "eer_percent": 7.4654
  },
  "asr_audit": {
    "text": "示例转写文本",
    "risk": "待 LLM 分析"
  },
  "speaker_match": {
    "target": "target_speaker",
    "similarity": 0.92
  },
  "vision_audit": {
    "summary": "画面审计结果"
  },
  "timestamp": {
    "sha256": "..."
  }
}
```

运行：

```bash
venv/bin/python extension_audit.py report \
  --input artifacts/audit/demo_video/audit_payload.json \
  --output artifacts/audit/demo_video/final_report.json
```

需要：

```text
.env 中已配置 DeepSeek API Key
```

## 四、推荐验证顺序

建议按下面顺序验证，避免一开始就卡在最重依赖上：

1. FFmpeg 音画分离；
2. 本地模拟存证；
3. Whisper ASR；
4. 文本风险审计；
5. 综合风险报告；
6. WeSpeaker 声纹注册与匹配；
7. VLM 图像审计；
8. Milvus 服务化声纹库。

最低可演示闭环：

```text
视频 -> 音频 -> ASR -> 文本风险审计 -> 本地存证 -> LLM 综合报告
```

完整作品化闭环：

```text
视频 -> 音画分离 -> ASR -> 文本审计
     -> 关键帧 -> VLM 图像审计
     -> 音频 -> SLS 伪造检测
     -> 音频 -> WeSpeaker 声纹匹配
     -> 本地/正式存证
     -> LLM 综合风险报告
```

## 五、不要在主环境执行的命令

不要在当前主 `venv` 里执行：

```bash
pip install git+https://github.com/wenet-e2e/wespeaker.git
pip install pymilvus
```

原因：

- WeSpeaker 会尝试升级 `torch/torchaudio`；
- pymilvus 会尝试升级 `protobuf`；
- 这两项都可能破坏当前 SLS 主模型复现环境。

如必须安装，请使用独立环境。
